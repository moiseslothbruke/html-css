conceitos

BRANCH: É uma linha de desenvolvimento paralela à principal. Serve para trabalhar em novas funcionalidades ou correções sem afetar o código principal (geralmente a branch main ou master).

Issue (problema ou tarefa): É um registro de algo a ser feito, corrigido ou discutido no projeto. Pode representar um bug, uma melhoria ou uma nova funcionalidade.

Commit (comprometimento): É o salvamento de mudanças no código com uma descrição. Ele registra um ponto da história do projeto com as alterações feitas.

Push (empurrar): É o envio dos commits locais (do seu computador) para o repositório remoto (ex: GitHub), atualizando o projeto online.

Pull (puxar): É o contrário do push: traz as mudanças do repositório remoto para o seu computador, mantendo seu código atualizado com o que os outros fizeram.

Redes sociais

O github funciona como uma rede social, de forma simples, é possível se comunicar e trocar conhecimento através dos repositórios.
